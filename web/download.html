Debian/Ubuntu packages

Use the following debian apt repository to get access to latest Vortex Library 1.1 packages:

NOTE: do not use these sources if you are using Core-Admin unless you know what you are doing.

    # public aspl ubuntu repository (XENIAL)
    deb http://www.aspl.es/ubuntu/public xenial/
 
    # public aspl ubuntu repository (PRECISE)
    deb http://www.aspl.es/ubuntu/public precise/
 
    # public aspl debian repository (JESSIE)

    # public aspl debian repository (WHEEZY)

    # public aspl debian repository (SQUEEZE)

    # public aspl debian repository (Raspbian Wheezy)

Latest CentOS 6/7 packages

CentOS packages are also provided using the following yum repository:

NOTE: do not use these sources if you are using Core-Admin unless you know what you are doing.


  # /etc/yum.repos.d/ASPL-Base.repo
  # Aspl-Public repo 
  #
  # ASPL packages for libaxl, vortex, nopoll, myqtt, ext-dns, valvula
  # and other open source public projects
  #
  
  [aspl-public]
  name=Aspl-Public-$releasever - Base
  baseurl=http://www.aspl.es/centos/public/$releasever/$basearch/
  gpgcheck=0
 
  # Aspl-Private repo 
  #
  # ASPL packages for commercial projects like core-admin
  #
  [aspl-private]
  name=Aspl-Private-$releasever - Base
  baseurl=http://www.aspl.es/centos/private/$releasever/$basearch/
  gpgcheck=0

Then run an yum search nopoll to search noPoll packages.

FreeBSD packages

FreeBSD ready to use packages are also provided using the following pkg repository:

NOTE: do not use these sources if you are using Core-Admin unless you know what you are doing.

 # FreeBSD 10.X
 ASPL: {
   url: "pkg+http://www.aspl.es/freebsd/public/10/${ABI}",
   mirror_type: "srv",
   enabled: yes
 }

 # FreeBSD 9.X
 ASPL: {
   url: "pkg+http://www.aspl.es/freebsd/public/9/${ABI}",
   mirror_type: "srv",
   enabled: yes
 }

Then run an pkg search vortex to search Vortex Library packages.

Getting source code from the SVN repository (github.com)

Although previous sections contains references to current stable releases, you can also check out current latest Vortex Library status at this time.

If you find something missing or not properly implemented, you can either drop us an email at the Vortex mailing list or test current SVN status.

Check the following url to get access the latest source code:

Current stable 1.1 series SVN and GitHub:
svn co https://github.com/ASPLes/libvortex-1.1
git clone https://github.com/ASPLes/libvortex-1.1
Old stable 1.0 series SVN:
https://dolphin.aspl.es/svn/publico/af-arch/trunk/libvortex/
